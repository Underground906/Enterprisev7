## Purpose

The Working Practices pillar defines **how work gets done** in Enterprise_OS. It contains session design patterns, sustainable pacing protocols, AI guidance, and the milestone-based workflow system that structures all execution.

**Core Innovation:** The Milestone Anchor Loop — a 90-120 minute structured work cycle that ensures consistent progress with visible wins.

## Key Frameworks

### 1. The 7 Milestones
A dependency-ordered system of workflow milestones:
- M1: RAG Database & Knowledge Base
- M2: Agent Handoff
- M3: Best Practices & Session Hygiene
- M4: Big Picture PRD
- M5: Claude Analysis
- M6: Folder Structure Correction
- M7: Project Goals & Context

### 2. Milestone Anchor Loop (90-120 min)
```
INVENTORY → EXTRACT → SYNTHESIZE → DERIVE → FILE → LOG
```

### 3. Three-Lane Ingestion
- Lane A: Historical Backlog
- Lane B: Fresh Real-time
- Lane C: External Media

### 4. Two-Hour Work Blocks
- ≤3 decisions per cycle
- Visible win required
- One tiny next step always defined

### 5. Session Hygiene
- Capture every Q→A turn
- Two-lane staging (review vs auto)
- Manifest-first discipline

### 6. Agent Handoff Protocol
- Micro-step execution
- ≥0.90 confidence threshold
- Always dry-run first

## Quick Start

1. **Choose milestone focus** for your session
2. **Start Anchor Loop:** Inventory files, identify gaps
3. **Extract:** Capture decisions with evidence
4. **Synthesize:** Create anchor document
5. **Derive:** Spawn SOPs, templates
6. **File:** PARA route, update indices
7. **Log:** DECISIONS.md, set next step

---

## Critical Rules

- **Max 3 decisions per 2-hour block** (avoid fatigue)
- **Always define next tiny step** (maintain momentum)
- **60-second reset** when overwhelmed
- **Evidence-based finals only** (no assumptions)
- **Dry-run before any automated operation**
